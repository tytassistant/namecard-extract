Connect to Microsoft
Sign in with your Microsoft account to sync extracted namecards to Microsoft Contacts. Your session stays active for up to 90 days.
  1. Go to Azure AD > App registrations
  2. Click New registration, name it (e.g. “Namecard Extract”)
  3. Under Supported account types, choose Accounts in any org & personal
  4. Under Redirect URI, select Single-page application (SPA) and enter this page’s URL
  5. Click Register, then copy the Application (client) ID
  6. Go to API permissions > Add > Microsoft Graph > Delegated > add Contacts.ReadWrite and User.Read
Long-lived sessions
MSAL.js caches refresh tokens in localStorage. Once signed in, your session stays active for up to 90 days without re-login.
Found in Azure AD > App registrations > Overview
Connect to Google
Sign in with your Google account to sync extracted namecards to Google Contacts. Note: the session token lasts 1 hour — you'll need to sign in again each visit.
  1. Go to Google Cloud Console > APIs & Services > Library and enable the People API
  2. Go to OAuth consent screen > External > add scope https://www.googleapis.com/auth/contacts
  3. Go to Credentials > Create OAuth Client ID > Web application > add this page’s URL under Authorized JavaScript origins
  4. Copy the Client ID (ends in .apps.googleusercontent.com) and paste it below
Short-lived tokens
Unlike Microsoft, Google tokens last only 1 hour and are not stored across page refreshes. You’ll be prompted to sign in again on each visit. Your Client ID is saved so you won’t need to re-enter it.
Found in Google Cloud Console > Credentials
